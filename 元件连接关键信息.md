## 一、主控芯片 ESP32‑S3（开发板 ESP32‑S3‑DevKitC‑1）

### 1. I2 C 总线（与 MCP23017 通信）

```c
SCL  → GPIO48
SDA  → GPIO21
```

- 板载 **​4.7 kΩ 上拉到 3.3 V​**，初始化 I2 C 时无需外接上拉电阻。

- 在 Arduino 框架下使用 `Wire.begin(21, 48)` 即可。

### 2. I2 S 音频输出（至 PCM5102 DAC）

```c
BCK  ( 位时钟 )  → GPIO4
LRCK ( 左右时钟 ) → GPIO5
DIN  ( 数据 )    → GPIO6
```

- PCM5102 工作在 **​硬件模式​**，无需 I²C 配置，只需 ESP32 端初始化 I2 S 并连续发送 PCM 数据。

### 3. ADC 输入（电位器）

```c
VR1 滑动端 → GPIO7
```

- 电位器两端分别接 3.3 V 和 GND，模拟电压 0～3.3 V，直接通过 `analogRead(7)` 读取（12 位 ADC 分辨率）。

### 5. 外部中断（来自 MCP23017）

```c
INTA → GPIO13
```

- MCP23017 的 INTA 引脚为开漏输出，**​板载 10 kΩ 上拉到 3.3 V​**。

- 可配置 ESP32 的 `GPIO13` 为上升沿 / 下降沿中断（低电平有效表示有事件发生），减少 I2 C 轮询消耗。

## 二、I/O 扩展器 MCP23017（I2 C 地址 0x20）

**​地址确定：​** A0、A1、A2 均接地 → 7 位地址 `0x20`

| 引脚 | 网络名称 | 功能 | 软件配置 |
||-||-|

| GPB2 | SW_B0 | 拨动开关 SW1 | 输入模式，**​使能内部上拉​**（开关闭合接 GND） |
| GPB3 | SW_B1 | 拨动开关 SW2 | 输入模式，使能内部上拉 |
| GPB4 | SW_B2 | 拨动开关 SW3 | 输入模式，使能内部上拉 |
| GPB5 | SW_B3 | 拨动开关 SW4 | 输入模式，使能内部上拉 |
| GPB6 | SW_B4 | 按钮 | 输入模式，使能内部上拉 |
| 其他 GPA0～7、GPB0/GPB7 | 未连接 | 闲置 | 可忽略或配置为输出低电平 |

**​读取示例：​**

```cpp
Wire.beginTransmission(0x20);
Wire.write(0x13);    // GPIOB 寄存器
Wire.endTransmission();
Wire.requestFrom(0x20, 1);
uint8_t portb = Wire.read();  // bit1 = 耳机，bit2~6 = 开关
```

**​中断使用：​**

若使用硬件中断，需使能 MCP23017 的 GPB 引脚中断，并将 INTA 映射到 ESP32 的 `GPIO13` 以触发读取。

## 三、DAC 模块 PCM5102（纯硬件）

- **​无需任何软件控制​**，上电即工作。

- 使用的硬件接线：

`VCC` → 3.3 V，`GND` → GND

`FLT`/`DMP`/`FMT` 已接地，`XMT`（静音）已接 3.3 V（不静音）。

- 编程时只需关注 I2 S 接口的 GPIO4/5/6。

## 四、耳机放大器 MAX97220

- `SHDN#` 悬空，芯片默认使能，**​不需要代码控制​**。
